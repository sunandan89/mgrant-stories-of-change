Feature Concept

Stories of Change

A central impact storytelling module for mGrant — where NGOs capture ground-level stories and comms teams curate them across all projects, without diving into individual grants.

Platform: mGrant (Frappe v16) Date: 16 April 2026 Status: Spec v2 — Decisions Locked

1 The Problem

Today, impact evidence is scattered across three unconnected surfaces inside a Grant:

Exists

Field Visit Report

Structured compliance tool for PMs. Checklist-driven. 0 records on staging — underused or recently added. Lives in "LIC" module.

Exists

File Section (Attachments)

Generic Frappe file attachments on the Grant form (MoU, reports). No narrative, no categorization as "impact content". Already synced to Document Registry.

New

Stories of Change

Does not exist yet. NGOs have no dedicated place to share impact narratives, photos, or videos. Comms team has no cross-project story feed.

The comms team's core pain: they must log into each project's grant to find usable content. There's no central feed, no way to search by theme, geography, or media type, and no curation workflow.

2 Recommendation: Keep Them Separate

Verdict: Three distinct things, one unifying hub.
Field Visit Report, File Section, and Stories of Change serve different audiences, different purposes, and have different data shapes. Merging them would make each worse. Instead, build Stories of Change as its own DocType and give the comms team a central browsing experience — the same pattern as the Document Registry.

Why not merge with Field Visit?

DimensionField Visit ReportStories of Change
PurposeMonitoring & complianceCommunications & impact showcase
Primary creatorPM / SPM (internal)NGO partner + anyone on the grant
Data shapeStructured checklist, scores, observationsFree-form narrative + rich media gallery
ConsumerGrant manager reviewing deliveryComms team picking stories for reports, website, social
Approval needInternal sign-offLight curation (featured / not featured)
FrequencyTied to visit scheduleAnytime — whenever something noteworthy happens
Decision: New "Stories of Change" tab on Grant form. It will sit after the Field Visit tab. This keeps the Grant form organized without hiding stories inside another tab. The tab will show linked stories with an "+ Add Story" button and a "View All" link to the central hub.

CSR Manager ✓ ✓ ✓ ✗ — Full read across all stories. Can feature/archive.
Comms User (new role) ✓ Curation tab only ✗ ✗ — Read all stories. Can only edit curation fields (permlevel 1). Cannot modify the story itself.
Donor User ✓ ✗ ✗ ✗ — Read-only for stories in their funded projects. Filtered by donor field.
Comms User role: This is a new, lightweight role. The comms team doesn't need to understand grants or budgets — they just need to browse stories, mark favourites, and note where stories were used. The Curation tab fields sit at permlevel: 1, so only roles with level 1 write access can edit them.
Cross-NGO visibility (peer learning): NGO users can see Submitted and Featured stories from all NGOs — not just their own. This encourages peer learning and inspiration across the partner network. Draft stories remain private to the creator. Write and delete permissions are restricted to own stories only (if_owner). The permission_query_conditions hook enforces this: (owner = '{user}') OR (status IN ('Submitted', 'Featured')).

6 Central Stories Hub — The Comms Experience

This is the key differentiator. The comms team accesses "Stories of Change" from the main sidebar — not from inside any grant.

How it works (native Frappe, zero custom UI)

  1. Workspace sidebar entry: "Stories of Change" appears under the mGrant app sidebar, pointing to the Story of Change List View.
  2. List View with standard filters: Filter by NGO, project/donor, theme, story type, geography (state/district), status, date range.
  3. Cover image thumbnail: The image field renders as a thumbnail in the list — comms can visually browse.
  4. "Featured" quick filter: One click to see only featured stories.
  5. Report Builder: Comms team can build custom reports (e.g., "all featured stories from Maharashtra in FY 2025-26").
No custom dashboard needed for v1. Frappe's native List View + filters + Report Builder gives the comms team everything they need. A visual gallery dashboard (card layout with cover images) can be a v2 enhancement using a custom Page.

Comparison with Document Registry approach

The key difference: Document Registry is a mirror (sync from files). Stories of Change is a source (created directly here). Simpler architecture — no hooks needed.

7 Grant-Side Integration — New Tab

Stories of Change gets its own dedicated tab on the Grant form, placed after the Field Visit tab. This keeps the tab bar logical: monitoring activities (Field Visit) → impact evidence (Stories of Change) → files.

Implementation approach

Since Story of Change is a separate DocType (not a child table), the tab is implemented via a Custom Tab Break + HTML field on the Grant form. A client script fetches linked stories and renders a summary list with action buttons.

Grant Form Tab Bar: ┌────────┬────────┬─────┬─────┬─────────────┬══════════════════┬───────┐ │Details │Budget │ LFA │ ... │Field Visit │ Stories of Change │ Files │ └────────┴────────┴─────┴─────┴─────────────┴══════════════════┴───────┘ Tab contents (HTML rendered by client script): ┌──────────────────────────────────────────────────────────────┐ │ [+ Add Story of Change] │ │ │ │ ┌────────┬──────────────────────────────┬──────────┬───────┐ │ │ │ Date │ Title │ Status │ │ │ │ ├────────┼──────────────────────────────┼──────────┼───────┤ │ │ │ 12 Mar │ Sunita's journey to fina... │ Featured │ Open →│ │ │ │ 28 Feb │ Community water project t... │ Draft │ Open →│ │ │ │ 15 Jan │ Youth training programme... │ Submitted│ Open →│ │ │ └────────┴──────────────────────────────┴──────────┴───────┘ │ │ │ │ Showing 3 of 3 stories │ [View All in Stories Hub →] │ └──────────────────────────────────────────────────────────────┘

Technical details

  • Add a Custom Tab Break on the Grant DocType labelled "Stories of Change", inserted after the Field Visit tab.
  • Inside the tab, place a Custom HTML field (stories_of_change_html).
  • A Grant client script calls frappe.call to fetch stories linked to the current grant, renders the list in the HTML field, and wires up the "+ Add Story" button to frappe.new_doc('Story of Change', { grant: frm.doc.name }).
  • The "View All" link opens the Story of Change List View pre-filtered to the current grant.
Voluntary, not mandatory. There is no requirement for NGOs to submit a minimum number of stories. The tab will simply show "No stories uploaded yet" with the add button if empty.

Decision: Rename Gallery → Story of Change. Since Gallery has 0 records and was originally meant for photo/document uploads (a purpose now served by the Files section), it will be cleanly repurposed via Frappe's rename_doc. This reuses the existing DB table and keeps git history clean.

Migration steps

  1. Rename the DocType from "Gallery" to "Story of Change" (Frappe's rename_doc handles the DB table rename).
  2. Update autoname from FILE-{######} to SOC-.YYYY.-.####.
  3. Keep the existing 4 fields and repurpose them:
    • title → stays as-is (story title)
    • image → rename to cover_image, change to Attach Image
    • document_type + related_to → remove (replaced by direct Grant link)
  4. Add all new fields per the spec above (narrative, grant, beneficiary, media child table, classification, curation).
  5. Update permissions to match the new role matrix.
  6. Set max file size for video uploads to 50 MB (via site config or validation in controller).

8 Connecting Field Visit → Story of Change

Even though we keep them separate, there's a natural workflow: a PM conducts a field visit and wants to capture a story from it.

Solution: "Create Story" button on Field Visit Report

Add a custom button to the Field Visit Report form toolbar: "Create Story of Change". When clicked, it opens a new Story of Change form pre-filled with:

  • Grant (from the field visit)
  • Story date = field visit date
  • A link back to the field visit in the narrative (optional)

This is a one-liner in the Field Visit Report client script:

frm.add_custom_button('Create Story of Change', () => { frappe.new_doc('Story of Change', { grant: frm.doc.grant, story_date: frm.doc.visit_date }); }, __('Actions'));

This keeps both DocTypes clean while providing a smooth workflow bridge.
